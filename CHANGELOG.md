# Changelog

## 0.5.0 - 2026-07-14

- 上下控制改为拇指与食指捏合后上下拖动的连续滚动；移动手向上，页面向上，松开即停。
- 张开手掌只保留左右挥动翻页，避免上下挥动与捏合滚动冲突。
- 捏合时在摄像头骨架上显示拇指尖与食指尖之间的黄色定位球。

## 0.4.4 - 2026-07-14

- 网页右侧提示改为固定 88×88 的黑色正方形，不再自动缩成难以辨认的竖条；动作触发时显示更大的绿色方向箭头。
- 后台识别以每秒 15 次把本机手部关键点状态同步给控制窗口，实时预览重新显示 21 点骨架连线。

## 0.4.3 - 2026-07-14

- 后台 HandTracker 改用独立定时轮询，不再依赖 Offscreen Document 里可能不触发的视频或动画帧回调。
- 控制窗口会显示“已检测到手掌”或“正在寻找张开的手掌”，便于分辨识别层与网页动作层的问题。

## 0.4.2 - 2026-07-14

- 修复后台识别已经运行时重新打开控制窗口只显示黑色状态页的问题：新窗口会自动重新连接实时预览。

## 0.4.1 - 2026-07-14

- 控制窗口恢复实时摄像头预览；Chrome 已有授权时不会重复弹出权限窗口，这是正常行为。
- 控制窗口在鼠标仍停留其中时保持展开；只在鼠标移出后收起。

## 0.4.0 - 2026-07-14

- 把摄像头和 HandLandmarker 迁入 Chrome `offscreen` 后台页面；控制窗口失焦、隐藏或关闭后，识别仍持续运行。
- 手势取帧改用摄像头视频帧回调，避免依赖前台窗口动画帧。
- 在受控网页右侧注入黑色状态提示条；点击网页不会让提示消失，识别动作时在页面内闪对应绿色箭头。

## 0.3.2 - 2026-07-14

- 紧凑标签改为悬停自动展开；鼠标离开约半秒或停留 5 秒后自动收回，不再需要点击。
- 紧凑标签平时保持纯黑；仅在识别动作时，短暂显示对应的绿色大箭头。

## 0.3.1 - 2026-07-14

- 摄像头启动 5 秒后，控制窗口自动缩成屏幕右侧的窄标签，手势识别持续运行。
- 点击窄标签可展开控制画面；全尺寸窗口新增手动“收起”按钮。

## 0.3.0 - 2026-07-14

- 替换 Chrome Side Panel 为独立手势控制窗口，绕开 macOS 上 Side Panel 无法弹出摄像头授权的限制。
- 插件图标点击时先绑定当前网页，再打开控制窗口；摄像头与网页动作都保持关联到原网页。

## 0.2.1 - 2026-07-14

- 修复 Side Panel 先连接网页、失败后不请求摄像头的启动顺序问题。
- 现在摄像头和本机手势模型会先启动；网页未连接时显示明确提示，不再阻断摄像头授权。

## 0.2.0 - 2026-07-13

- 新增 Chrome Manifest V3 手势浏览插件与常驻 Side Panel。
- 新增四方向离散挥动检测、650ms 冷却、稳定/离手重新激活和轨迹测试。
- 上下挥动滚动 75% 视口，左右挥动发送网页 ArrowLeft / ArrowRight 翻页动作。
- 新增网页动作适配层、输入框保护、最小权限和错误状态提示。
- 新增浏览动作验收页，以及独立的 Web Demo / Extension 构建目标。
- 验证：`npm run typecheck`、`npm test`、`npm run build`、真实 Chromium 测试页。

## 0.1.0 - 2026-07-13

- 建立黑洞手势控制独立 Web Demo。
- 完成 Three.js 无限循环黑洞、HandLandmarker 单手追踪、21 点骨架、左右手颜色区分。
- 完成手掌位置转向、手掌倾斜持续旋转、距离调速、张手继续和握拳暂停。
- 完成 MediaPipe 资源同步脚本，运行时从本项目同源加载 WASM 和模型资源。
- 验证：`npm run typecheck`、`npm test`、`npm run build`。
