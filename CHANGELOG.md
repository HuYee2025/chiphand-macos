# Changelog

## macOS 0.6.0 - 2026-07-15

- 主控制窗口新增“控制手”单选控件，默认右手，可切换为左手并持久保存。
- 所选手之外的关键点在进入手势引擎前被严格丢弃，不能触发捏合、翻页、返回或点赞；切换时立即取消上一只手的活动状态。
- MediaPipe 继续保持 `numHands: 1`，不增加双手推理负担；校准窗口也不绘制未选择手的骨架。

## macOS 0.5.1 - 2026-07-15

- 取消展开状态条的自由拖动，修复窗口随 SwiftUI 全局拖动坐标反复重算造成的颤动。
- 保留右侧三道杠，单击直接收到屏幕右边缘；迷你窗口采用左圆右方造型，只能通过 AppKit 绝对坐标上下移动，双击恢复默认位置。
- 双击展开反馈条可暂停/恢复识别；暂停时停止摄像头、MediaPipe 和系统事件，保留红色状态圆点与“已暂停手势控制”提示。

## macOS 0.5.0 - 2026-07-15

- 将全屏骨架与实时状态条拆成两个独立浮层：骨架继续完全穿透鼠标，状态条可拖拽且不抢前台 App 焦点。
- 状态条拖到屏幕左右边缘会自动吸附收起，只保留可变色的状态圆点；点击或向屏幕内拖动可再次展开。
- 记住状态条位置、吸附边和收起状态；默认位置仍保持在 Dock 上方。

## macOS 0.4.1 - 2026-07-15

- 修复辅助功能开关表面开启但 App 实际未获授权的循环：本地 ad-hoc 签名加入稳定 designated requirement，使后续构建继续匹配同一 GestureControl 权限身份。
- 首次从旧签名升级需要重置一次 GestureControl 的旧辅助功能记录；不影响其他应用权限。

## macOS 0.4.0 - 2026-07-15

- MediaPipe 主引擎升级为 Gesture Recognizer，保持 21 点骨架并原生识别 `Victory` 与 `Thumb_Up`；单手上限改为 1，停止无效的第二只手计算。
- 新增 V 手势稳定后向左挥动返回上一页，仅对 Chrome、Safari、Edge 和夸克发送 `Command + [`；静止 V、右挥或非浏览器不执行。
- 新增竖起拇指点赞识别与全屏点赞标记，目前仅显示测试状态，不点击网页、不产生真实点赞。
- 捏合改为严格 OK 手势：食指与拇指尖接触，同时中指、无名指、小指必须张开，解决握拳和无意晃动误触滚动。
- 新增推理耗时、实时 FPS 和 30/24 FPS 自适应调度；Apple Vision 备用模式明确标注新手势不可用。

## macOS 0.3.0 · ⭐ 重要稳定版 - 2026-07-15

- 主识别引擎从 Apple Vision 切换为本机 MediaPipe Hand Landmarker：21 点、左右手、最多双手检测、GPU 优先，Apple Vision 仅在 MediaPipe 启动失败时备用。
- 内置仅监听 `127.0.0.1` 的静态资源服务，在 WKWebView 中离线运行 WASM 与模型；摄像头画面和关键点不上传。
- 校准画面改为自拍镜像；视频、校准骨架和全屏 HUD 同步水平镜像，显示层简化为腕点和每指 3 段，底层仍保留完整 21 点。
- 捏合时在拇指与食指中点显示黄色圆点，松开立即消失；状态动态显示左右手、张掌、握拳、单指、捏合和翻页结果。
- 系统滚动改为 HID event tap 全局注入，并把事件位置放在目标窗口中心，修复 Chromium 忽略 PID 定向滚轮事件的问题。
- App 改为正常 Dock 应用并保留菜单栏入口，避免菜单栏项目被 macOS 隐藏后启动即退出，也解决退出后找不到入口的问题。

## 1.0.1 - 2026-07-14

- 修复 1.0.0 原生控制窗口仍沿用 350px 旧高度，导致常驻“高级设置”按钮落在窗口裁切区、实际无法看见的问题；默认窗口增至 390px，高级设置入口完整可见。

## 1.0.0 - 2026-07-14

- 作为首个可交付版本，新增 `npm run package:extension`，生成可提交 Chrome Web Store 与 Microsoft Edge Add-ons 的标准 ZIP 包。
- “高级设置”现在始终显示在摄像头开关下方；展开后保留两条灵敏度滑杆与“显示手部网格 / 显示捏合控制圆点”两个开发者开关，不再依赖摄像头是否正在运行。
- 控制窗口只在鼠标真正移出窗口范围后收起；删除窗口 `blur` 的 120ms 关闭路径，点击网页或切换焦点不会再抢先关闭预览。

## 0.11.0 - 2026-07-14

- 网页默认不再显示捏合圆点，阅读时保持干净；反色圆点与 21 点手部网格移入摄像头启动后的“高级模式”，两个开关默认关闭，仅作开发调试使用。
- “高级模式”紧跟“开启/关闭摄像头”按钮，只在摄像头运行时显示；保留左右挥手、手指捏合两条即时生效的灵敏度滑杆。
- 设置扩展为本机持久化的四项偏好，并在网页脚本新注入、控制窗口关闭恢复时同步，避免开关状态丢失。

## 0.10.0 - 2026-07-14

- 修复网页捏合点大多显示白色：移出 Shadow DOM，改用直接作用于网页底图的 `backdrop-filter: invert(1)`，圆内显示其下方实际画面的反色。
- 修复关闭控制窗口后网页偶发失效：原生窗口关闭时，Service Worker 会重新确认网页接收器、恢复入口并同步后台识别状态；预览窗口关闭不再成为动作链路的断点。
- 新增可折叠“高级设置”：可分别微调左右挥手、手指捏合灵敏度，默认 `50 · 标准` 保持原阈值；设置保存在本机扩展存储，立即同步给后台识别，不必重启摄像头。

## 0.9.1 - 2026-07-14

- 网页捏合点改为与右侧入口同为 28px 的纯色圆，不再固定绿色、无投影；通过 CSS `mix-blend-mode: difference` 自动呈现网页底色的反色。
- 控制窗口改为 300×350；右缘贴齐 Chrome 内容区，入口黑点下移并在展开时隐藏，不再与窗口并列露出。
- 启停控件恢复为满宽圆角文字按钮：`开启摄像头` / `关闭摄像头`，删除播放器式符号与多余底部留白。

## 0.9.0 - 2026-07-14

- 捏合阈值从归一化距离 `0.34` 收紧到 `0.18`：只有拇指食指近乎贴合才出现控制点，稍微分开即消失；绿色点改为不透明、直径 36px，并保留极轻投影。
- 手部状态同步提升到 30 FPS，网页端以 `requestAnimationFrame` 插值绘制绿色点；同时降低关键点平滑带来的拖尾，减少跟手卡顿。
- 控制窗口收紧为 300×400、16:9 视频和圆形播放/停止按钮；打开时自动贴近受控网页右侧小黑点。小黑点在窗口打开/关闭时缩放淡入淡出，形成展开/收起的视觉过渡。

## 0.8.0 - 2026-07-14

- 网页层改为极简反馈：捏合时只显示半透明绿色中点圆，不再绘制手部网格、黄点特效或黑色动作面板；松开或丢手后圆点立刻消失。
- 左挥改为平滑上滚并显示中央空心 `↑`，右挥改为平滑下滚并显示中央空心 `↓`；移除合成 `ArrowLeft` / `ArrowRight` 翻页路径。
- 网页入口缩为右侧 28px 黑色圆点；控制窗口移除冗余动作屏，只保留摄像头预览、状态和启动/停止。

## 0.7.2 - 2026-07-14

- 修复控制窗口显示 `Receiving end does not exist` 后网页无响应的问题：网页控制脚本现在是独立单文件注入，注入后必须 ping 确认接收器存在。
- 页面刷新或脚本被销毁后，下一次手势会自动重新注入并只重试一次；正常运行期间不再为每一帧手势额外 ping，避免拖动滚动被消息检查拖慢。

## 0.7.1 - 2026-07-14

- 修复网页手部网格启用后捏合滚动和左右翻页失效的问题：Content Script 现在只处理 Service Worker 明确转发给网页的消息，不会把 Offscreen 后台的识别状态和动作请求误当成网页动作。

## 0.7.0 - 2026-07-14

- 反转捏合拖动的页面滚动方向：手向上拖时页面内容向上移动，因此能查看下方内容，符合触控板和触屏的直接操作习惯。
- 控制窗口关闭后，网页直接显示镜像映射的手部 21 点网格；捏合时拇指食指中点显示为黄色指针球，持续跟随页面上的对应位置。
