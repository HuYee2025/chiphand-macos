# 生产蓝图

## 项目一句话

使用 MacBook 摄像头和单手挥动控制 Chrome 网页滚动与左右翻页。

## 目标用户/使用场景

- MacBook Chrome 网页阅读、在线文档、相册和支持方向键的阅读器。
- 第一版只支持 Chrome 114+ 当前活动标签页。
- 黑洞 Web Demo 继续作为手部追踪与性能验证场。

## 核心体验

- 张开手掌快速向四个方向挥动，一次只触发一个动作。
- 上下挥动平滑滚动约 75% 视口；左右挥动触发网页上一页/下一页。
- 650ms 冷却，手掌稳定或离开画面后重新激活，避免重复触发。
- 独立控制窗口显示摄像头、21 点骨架、识别状态和最近动作。

## 功能/内容结构

- Chrome Manifest V3 扩展、独立控制窗口、Service Worker 和动态网页控制脚本。
- 通用网页动作适配器，以及后续增加站点适配器的注册表。
- 自建验收页验证滚动、左右翻页和输入框保护。
- 原有 Three.js 黑洞 Demo 保留，不与插件运行入口耦合。

## 视觉与风格

- 控制窗口使用深色、克制的本地实验工具风格。
- 摄像头和方向状态优先，避免装饰抢占狭窄侧栏空间。

## 技术约束

- 技术栈为 Vite、TypeScript、Chrome MV3 和 MediaPipe Tasks Vision。
- 插件模型、WASM 和摄像头推理完全本地运行。
- HandLandmarker 自适应 24–30 FPS，GPU 优先、CPU 备用。
- 关键点逐点平滑，短暂遮挡保留最后可信姿态约 220ms。

## 不做什么

- 第一版不控制 macOS 全局键盘，不开发 Mac 菜单栏应用。
- 不控制浏览器历史前进/后退。
- 不承诺所有网站接受合成的左右方向键。
- 不申请 `<all_urls>` 永久权限，不上传摄像头画面。
- 暂不训练自定义手势模型。
