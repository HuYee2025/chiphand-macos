# 生产蓝图

## 项目一句话

使用 MacBook 摄像头和单手手势，在 macOS 前台应用中进行系统级翻页和连续滚动。

## 目标用户/使用场景

- MacBook 上的 Chrome、Safari、Preview、Notion 等阅读场景。
- 第一版只控制当前前台应用，不识别屏幕内容，不移动鼠标光标。
- 浏览器插件与黑洞 Web Demo 冻结保留，继续作为历史验证场和回滚点。

## 核心体验

- 拇指与食指捏住约 0.08 秒后，上下拖动手掌连续滚动；手向上拖会查看下方内容，松开或丢手立即停止。
- 张开手掌向左/右快速挥动，分别向上/向下滚动约 75% 屏幕。
- 捏合有接触/释放阈值与跳变保护，左右挥动保持 650ms 冷却，避免重复触发。
- 菜单栏手掌图标承担常驻入口；窗口只在用户点击时展开，显示启停、权限、灵敏度和可选摄像头预览。

## 功能/内容结构

- 原生 SwiftUI 菜单栏 App、AVFoundation 摄像头、Apple Vision 手部关键点与 Core Graphics 系统滚动事件。
- 菜单栏提供启停、权限状态、灵敏度和调试预览。
- 系统事件只发给当前前台 App；切换 App 时取消正在进行的捏合。

## 视觉与风格

- 使用原生 macOS 菜单栏和 SF Symbols，不创建网页叠层。
- 原型验收期默认显示可关闭的置顶识别窗；验证完成后再把它改回按需开启。

## 技术约束

- 技术栈为 Swift、SwiftUI、AVFoundation、Vision、ApplicationServices 和 Core Graphics。
- 摄像头画面和关键点完全留在本机，不保存、不上传。
- 系统级输出必须获得摄像头和辅助功能权限；不申请屏幕录制权限。
- 系统控制比网页控制更严格：丢手不续帧，立即停止输出。

## 不做什么

- 不移动系统光标，不做鼠标点击、按住拖拽或 Air Mouse。
- 不发送合成键盘事件，不控制浏览器历史前进/后退。
- 不读取屏幕，不申请屏幕录制或输入监控权限。
- 暂不做登录启动、App Store 发布、自动更新或自定义模型训练。
