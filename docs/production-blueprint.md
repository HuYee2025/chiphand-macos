# 生产蓝图

## 项目一句话

使用 MacBook 摄像头和单手手势，在 macOS 前台应用中进行系统级翻页和连续滚动。

## 目标用户/使用场景

- MacBook 上的 Chrome、Safari、Preview、Notion 等阅读场景。
- 系统不识别屏幕内容；可选食指指针只在四个浏览器内移动真实鼠标，由网页自身完成悬停命中。
- 浏览器插件与黑洞 Web Demo 冻结保留，继续作为历史验证场和回滚点。

## 核心体验

- 拇指与食指捏住约 0.08 秒后先锁定移动方向：垂直锁定后上下拖动连续滚动，松开或丢手立即停止。
- 严格 OK 从左侧向右跨中线返回，从右侧向左跨中线前进；只允许 Chrome、Safari、Edge 和夸克执行。
- 张开手掌以手掌中心的短时水平轨迹翻页：向右挥动下翻，向左挥动上翻；达到水平距离即可，不强制跨过屏幕中线。
- 可选食指指针稳定后移动真实鼠标；停稳后保持食指不动，用拇指和中指轻捏执行单击，接触处显示第二黄点，点击位置冻结在最后稳定指尖。`0.8.3` 校准期间，接触黄点与点击待命解耦并显示实时距离。普通张掌仍负责翻页。V 手势只显示状态。
- 方向锁在松开前不会切换；小幅、斜向、近中线起手和未跨中线动作不执行。
- 捏合必须是严格 OK 手势：拇指食指接触，另外三指张开；握拳不会触发滚动。
- 竖起拇指只显示点赞识别状态，不执行点赞。
- 用户必须在右手和左手中选择一只控制手，默认右手；另一只手只会被忽略，不产生任何动作。
- Dock App 和菜单栏手掌图标都可作为入口；全屏透明 HUD 显示简化骨架和捏合位置，固定状态条可收成右边缘迷你条，双击可暂停/恢复识别，摄像头只作为可选校准窗口。

## 功能/内容结构

- 原生 SwiftUI App、内嵌 MediaPipe Gesture Recognizer（单手/21 点/左右手/内置手势/GPU 优先）、Apple Vision 备用与 Core Graphics HID 系统事件。
- 菜单栏提供启停、权限状态、灵敏度、控制手单选、全屏骨架 HUD 和摄像头校准窗口开关。
- 系统事件只发给当前前台 App；切换 App 时取消正在进行的捏合。

## 视觉与风格

- 使用原生 macOS 窗口、菜单栏、SF Symbols、点击穿透的骨架 HUD 和不抢焦点的可交互状态条，不修改网页内容。
- 原型验收期默认显示全屏骨架与底部状态；摄像头窗口默认关闭，只在校准镜像和误识别时打开。

## 技术约束

- 技术栈为 Swift、SwiftUI、WKWebView、MediaPipe WASM、AVFoundation、Vision、ApplicationServices 和 Core Graphics。
- 摄像头画面和关键点完全留在本机，不保存、不上传。
- 系统级输出必须获得摄像头和辅助功能权限；不申请屏幕录制权限。
- 系统控制比网页控制更严格：丢手不续帧，立即停止输出。

## 不做什么

- 不做鼠标按住拖拽或持续 Air Mouse；食指指针默认关闭，只支持主屏幕与四个浏览器。
- 除浏览器返回/前进所需的 `Command + [` 与 `Command + ]` 外，不发送其他合成键盘事件；导航只允许 Chrome、Safari、Edge 和夸克。
- 不读取屏幕，不申请屏幕录制或输入监控权限。
- 暂不做登录启动、App Store 发布、自动更新或自定义模型训练。
