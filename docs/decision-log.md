## 2026-07-12 独立 Web Demo

背景：
- 黑洞源码来自《火星先遣队》，当前项目另有手部识别参考截图。

决策：
- 在当前根目录建立独立应用，只复制黑洞思路和源码，不修改旧游戏项目。
- 第一版使用一只手；左右手功能相同，仅以红蓝区分。

原因：
- 降低旧游戏的复杂度，先验证摄像头控制手感和跨设备可行性。

影响：
- 当前项目拥有自己的依赖、测试、构建和部署入口。

## 2026-07-12 手势与性能方案

背景：
- 用户需要位置转向、张手继续、握拳暂停，并增加手掌左右旋转控制。
- MediaPipe 同步运行在主线程时，真实浏览器实测最低约 19 FPS。
- 3DXRguy 的 VIVERSE 源码调研显示其使用 `HandLandmarker`、GPU 优先、逐点平滑和短暂丢失续帧。

决策：
- 使用 `HandLandmarker` float16 模型和 21 个关键点，不使用 `GestureRecognizer` 内置分类。
- 手腕到中指根部的屏幕平面角度映射为虫洞旋转。
- 手掌倾斜控制的是旋转速度而非固定角度：保持倾斜会持续旋转，回正后停止。
- 张手、握拳和捏合由关节距离计算，收拇指不影响张手判断。
- 前进速度由手掌在画面中的相对尺寸控制：首次识别校准为 1×，靠近摄像头更快、远离更慢，并限制在 0.45×～2.4×。
- MediaPipe WASM 推理放入 ES Module Web Worker，自适应 24–30 FPS，GPU 优先、CPU 备用。
- 关键点逐点平滑；检测短暂失败时保持最后可信姿态 220ms。

原因：
- 关节距离判断不依赖拇指和内置标签；Worker 避免阻塞 Three.js 渲染。

影响：
- 摄像头开启后的黑洞渲染恢复约 60 FPS；模型和 WASM 从本项目同源加载。
- 极端 90° 侧面仍受单目 RGB 摄像头物理限制，但短时遮挡和拇指收拢的体验会明显改善。
