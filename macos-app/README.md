# macOS 系统手势控制原型

原生 Mac App 使用本机 MediaPipe Gesture Recognizer 识别手势，通过系统级滚动事件控制当前前台应用。Apple Vision 只作为启动失败时的备用引擎。

## 当前动作

- 严格 OK 手势捏住后上下移动：连续滚动，松开或丢手立即停止。
- 严格 OK 从屏幕左侧向右跨中线：在 Chrome、Safari、Edge 和夸克返回上一页。
- 严格 OK 从屏幕右侧向左跨中线：在上述浏览器前进下一页。
- 食指从左向右跨中线：向下翻约 75% 屏幕；从右向左跨中线：向上翻约 75% 屏幕。
- V 手势：只显示识别状态，暂不执行操作。
- 张开手掌：只显示识别状态，暂不执行操作。
- 竖起拇指：显示点赞识别反馈，不执行真实点赞。
- 不移动鼠标光标，不发送鼠标点击。
- 开启后默认在整个屏幕显示点击穿透的手部骨架和捏合点；底层仍使用完整 21 点，显示层每根手指简化为 3 段。
- 实时状态条固定在 Dock 上方默认位置；单击右侧三道杠收到屏幕右边缘，迷你条可上下移动，双击恢复。
- 双击展开状态条可暂停/恢复识别；暂停时圆点变红并显示“已暂停手势控制”。
- 摄像头校准窗口默认关闭，需要检查画面与骨架对齐时可从菜单打开。
- 主控制窗口可选择“右手”或“左手”，默认右手；只有所选手能显示骨架并执行动作。
- 菜单中的“测试系统下翻”可绕过手势识别，直接验证辅助功能和系统滚动是否正常。
- App 同时出现在 Dock/程序坞和菜单栏；关闭控制窗口不会停止后台手势识别。

## 开发

建议安装完整 Xcode，然后执行：

```bash
cd macos-app
swift test
swift run GestureControlCoreChecks
./scripts/build-app.sh
open build/GestureControl.app
```

稳定安装到“应用程序”：

```bash
./scripts/install-app.sh
```

首次启动需要允许“摄像头”和“辅助功能”。点击开启后如果辅助功能未生效，使用控制面板里的“设置”按钮；打开开关后 App 会自动继续启动，不需要反复点击。原型使用 ad-hoc 签名，重新构建并替换 App 后可能需要重新授权一次辅助功能。
