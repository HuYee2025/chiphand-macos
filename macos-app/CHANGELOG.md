# Changelog

## 1.1.1 - 2026-07-16

- 去掉重复的常驻主窗口，只保留菜单栏小手弹出的控制面板。
- 底部版本号改为读取应用实际版本，不再硬编码旧版本。

## 1.1.0 - 2026-07-16

- MediaPipe macOS 主识别器改为同时检测最多两只手，但只让用户选择的控制手进入手势引擎。
- 右手默认控制时，即使左手先进入或与右手同时出现在摄像头内，也会优先寻找右手，不再因为左手抢先出现而丢失右手。
- 左右手标签切换增加 `120ms` 稳定窗口；过渡期间不发送滚动、翻页、导航、指针或点击事件。
- 全屏骨架和校准画面只显示选中的控制手，另一只手保持完全无效。

## 1.0.1 - 2026-07-16

- 删除容易误解的“显示控制点”开关；食指黄色控制点、鼠标移动和拇指中指点击现在永久启用。
- 忽略旧版本保存的关闭状态，升级后无需再次进入设置；非浏览器、辅助功能未允许或 Apple Vision 备用模式会显示具体不可用原因。
- 食指状态不再出现“暂未设置操作”，受支持浏览器中改为“正在确认食指指针”。

## 1.0.0 - 2026-07-16

- 产品正式命名为“薯片手 / ChipHand”，启用固定 `com.huyee.chiphand` bundle ID 与全尺寸应用图标；菜单栏继续使用运行/暂停状态清楚的小手图标。
- 建立克莱因蓝、薯片金黄、白底黑线的视觉系统，重做主控制窗口的品牌头部、首次权限、状态、手势速查与显示控制卡片。
- 新增完全离线的网页版使用说明，包含安装、首次安全放行、权限、四种核心手势、反馈窗口、隐私和排障；内置四张现代线稿手势插图。
- App 内新增“使用说明 / 查看完整图解”入口，直接打开安装包内的离线网页。
- 构建改为 `arm64 + x86_64` Universal App；应用包内置图标、MediaPipe JavaScript/WASM/模型、说明文档、MIT License 与 Apache 2.0 第三方许可证。
- 新增 DMG/ZIP 打包与 SHA-256 校验脚本。免费开源版继续使用稳定 designated requirement 的 ad-hoc 签名，不依赖付费 Apple Developer 会员。

## 0.8.6 - 2026-07-16

- 中线反馈加长为 `352pt`，视觉改为清晰白色核心和分层蓝色外发光，触发时机仍与黄球首次触线同步。
- 主设置开关“食指指针（测试）”更名为“显示控制点”，不改变指针识别、鼠标映射或点击算法。
- 全屏反馈层与骨架显示解耦：关闭骨架只隐藏连线、白色节点、黄色捏合连线和点赞标记，黄色控制点与中线闪光继续显示；骨架选择会持久保存。

## 0.8.5 - 2026-07-16

- 水平导航方向锁成立后，首次到达或越过 x=50% 中线的同一帧立即输出导航事件，让黄色捏合球与蓝线闪光在视觉上同步。
- 删除方向锁后的 `14%` 重复最小位移等待；近中线起手保护、初始水平主导和每次捏合单次触发不变。
- 蓝色发光竖线从 `88pt` 加长到 `176pt`，持续时间与层次效果不变。

## 0.8.4 - 2026-07-16

- 严格 OK 导航捏合成功跨过屏幕中线时，在中线上显示 `88pt × 4pt` 蓝色发光竖线，纵向对齐黄色捏合球。
- 闪光采用蓝色核心、淡蓝外发光和轻微缩放淡出，总持续时间约 `240ms`，每次捏合最多触发一次。
- 非支持应用也显示跨线识别闪光，但仍不发送返回/前进按键；暂停、切换应用、丢手或关闭 HUD 会立即清除闪光。

## 0.8.3 - 2026-07-16

- 拇指中指轻捏直接使用严格 OK 的 `0.18` 接触阈值、`0.20` 释放阈值与 `80ms` 稳定时间，不再要求点击前先分开。
- 第二黄点不再依赖点击状态；拇指尖与中指尖的节点距离进入接触阈值即显示。
- 反馈文案附带实时归一化距离，接触时明确提示“已检测到拇指中指接触”，用于实机校准。
- 食指坐标映射、平滑和鼠标输出保持不变；持续捏住仍只消费一次点击，松开后才能开始下一次。

## 0.8.2 - 2026-07-16

- 点击改为稳定食指下的拇指中指轻捏：先分开 100ms 完成待命，接触 80ms 后在最后稳定 `indexTip` 位置发送一次普通左键。
- 指针激活后只要求食指伸直、无名指和小指收拢，中指与拇指可自由完成点击动作，原有指尖跟踪参数完全不变。
- 接触期间显示第二个黄色圆点；保持接触不会重复点击，必须分开后重新待命。

## 0.8.1 - 2026-07-15

- 点击组合手势改为稳定食指下张开拇指 120ms；食指资格不再依赖拇指收拢，展开过渡保持黄色点，点击使用最后稳定的 `indexTip`。
- 拇指保持张开只触发一次，收回至少 150ms 后才重新激活；张开整只手不再作为点击，继续保留左右挥动翻页。
- 左键事件改用 `.privateState` 并强制空 flags、左键编号和单击状态，确保发送的是无修饰键普通左键。

## 0.8.0 - 2026-07-15

- 新增浏览器白名单内的可选食指鼠标：严格 `Pointing_Up` 激活后使用镜像 `indexTip` 直接映射 MacBook 主屏幕，保留 180ms 分类容错并过滤小抖动与单帧跳变。
- 食指停稳后张开同一只手执行一次点击；该张掌被组合手势消费，不会同时触发翻页，未定位张掌只提示、不操作。
- 主菜单新增持久化测试开关，默认关闭；骨架与校准窗口新增黄色食指尖点，Apple Vision 备用模式不发送鼠标事件。

## 0.7.3 - 2026-07-15

- 恢复张开手掌左右挥动翻页：右挥下翻、左挥上翻，使用最近 `360ms` 的手掌中心轨迹，不要求跨过屏幕中线。
- 保留最小持续时间、水平主导、`650ms` 冷却和重新激活门槛，避免静止张手与单次长挥连续触发。
- 食指和 V 手势只显示状态；严格 OK 捏合滚动及返回/前进逻辑未改动。

## 0.7.2 - 2026-07-15

- 翻页距离改由食指尖 `indexTip` 计算，不再错误使用手掌中心；稳定阶段从首次姿态成立时保存指尖起点。
- 严格食指手势要求拇指、中指、无名指和小指收拢，仅食指伸直。
- 激活后容忍最多 180ms 分类波动，只要严格食指关键点仍成立就继续跟踪；超时或姿态散开立即取消。

## 0.7.1 - 2026-07-15

- 使用 MediaPipe `Pointing_Up` 替换 V 手势翻页：食指左向右跨中线下翻，右向左跨中线上翻。
- 沿用稳定时间、起点保护、水平主导、跨中线、单次触发和退出重激活规则，避免普通指向动作误触。
- V 手势保留为状态显示，不执行操作；严格 OK 捏合、浏览器导航和其他功能不变。

## 0.7.0 - 2026-07-15

- 严格 OK 捏合先判定并锁定方向：垂直移动连续滚动；从屏幕一侧横向跨过中线后执行浏览器返回或前进。
- V 手势改为左右跨中线翻页，左向右下翻、右向左上翻；离开 V 手势后才允许再次触发。
- 张开手掌改为只显示状态，不再发送翻页事件；浏览器导航继续受应用白名单保护。

## 0.6.0 - 2026-07-15

- 主控制窗口新增右手/左手互斥选择，首次启动默认右手并记住后续选择。
- 只有所选手能进入手势引擎；另一只手和未知左右手的数据会被忽略，切换时立即清除活动手势。
- 识别器维持单手推理，校准窗口隐藏未选择手的骨架，不增加双手计算量。

## 0.5.1 - 2026-07-15

- 展开状态条改为固定默认位置，右侧三道杠单击收起，删除会导致窗口颤动的自由拖动。
- 右边缘迷你条改为左圆右方，使用屏幕绝对鼠标坐标锁定水平位置；只能上下移动，双击恢复默认大小与位置。
- 新增反馈条双击暂停/恢复；暂停时圆点红色，显示“已暂停手势控制”，不保留手势或系统事件。

## 0.5.0 - 2026-07-15

- 全屏骨架和状态反馈拆成独立窗口：骨架保持鼠标穿透，状态条可拖动且不激活 App。
- 状态条拖到左右边缘会自动吸附收起，收起后只显示识别状态圆点；点击或向屏幕内拖动可展开。
- 保存用户的窗口位置、吸附边和收起状态，显示器变化后自动校正位置。

## 0.4.1 - 2026-07-15

- 修复辅助功能列表显示已开启、App 实际仍读取为未允许的问题：ad-hoc 签名现在包含固定 bundle identifier 的 designated requirement，不再只用每次构建变化的 CDHash 作为系统身份。
- 本次安装后需要清理一次旧 TCC 项并重新打开 GestureControl；后续本地小版本重建不再反复丢失同一授权。

## 0.4.0 - 2026-07-15

- MediaPipe Hand Landmarker 升级为 Gesture Recognizer，使用单手识别并输出 21 点、左右手、内置手势、置信度和推理耗时。
- V 手势稳定 220ms 后向左挥动，在 Chrome、Safari、Edge 和夸克执行返回上一页。
- 竖起拇指稳定 300ms 后显示点赞状态和标记，不执行网页点赞。
- 捏合改为严格 OK 手势：拇指食指接触且其余三指张开，避免握拳误判为捏住。
- 单手识别默认 30 FPS，高耗时时自动降到 24 FPS，控制窗显示实时 FPS 和推理耗时。

## 0.3.0 · ⭐ 重要稳定版 - 2026-07-15

- 首个稳定使用 MediaPipe GPU、本机离线运行时、HID 系统滚动、Dock 控制窗和全屏骨架 HUD 的版本。
- 回滚点保持为 `macos-v0.3.0`，不移动、不覆盖。

## 0.2.0 - 2026-07-15

- Apple Vision 骨架从 10 个采样点恢复为完整 21 点，并修正摄像头与骨架的水平坐标错位。
- 新增稀疏点数、完整手指链和点云跨度校验，过滤脸部附近的假手掌点云。
- 识别置信度从“所有点最低值”改为中位数，修复看得见手掌但手势引擎拒绝执行的问题。
- 新增点击穿透的全屏透明手势 HUD，默认显示 21 点骨架、捏合线和动态状态；摄像头窗口改为可选校准工具。
- 动态显示张开手掌、握拳、食指伸出、捏合确认、已捏合和翻页反馈。
- 新增“测试系统下翻”，用于独立验证辅助功能与系统滚动链路。

## 0.1.1 - 2026-07-15

- 修复辅助功能权限已经打开但 App 仍反复跳回系统设置的循环：启动按钮不再自动打开设置，App 每秒刷新权限，待授权生效后自动继续启动。
- 新增默认开启的悬浮识别测试窗，实时显示镜像摄像头、手部骨架、捏合连线和识别状态。
- 新增稳定安装脚本，将原型安装到 `/Applications/GestureControl.app`，避免退出后找不到入口。
- 新增标准 `XCTest` Target，覆盖捏合稳定、停止、翻页方向和取消状态。
- 更新本地构建号，安装新二进制后重新建立一次辅助功能授权，后续不重建 App 时不再反复失效。

## 0.1.0 - 2026-07-15

- 新增原生 macOS 菜单栏手势控制原型。
- 使用 AVFoundation 获取摄像头画面，使用 Apple Vision 检测单手关键点。
- 左挥上翻、右挥下翻约 75% 屏幕；捏合稳定 80ms 后上下拖动连续滚动。
- 通过 Core Graphics 将滚动事件发送给前台 App，不移动鼠标、不发送点击或键盘事件。
- 新增摄像头与辅助功能权限引导、两项灵敏度设置和可选调试预览。
- 新增纯 Swift 核心检查程序、本地 `.app` 打包与 ad-hoc 签名脚本。
